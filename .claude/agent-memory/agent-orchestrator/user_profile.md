The user is the founder and product owner of MindScrolling. Typically communicates requests in Spanish. Has a broad product vision and relies on the agent system to handle implementation details.

Prefers the Orchestrator to be proactive: analyze the request, identify all agents needed, and execute the full sequence without asking for clarification on standard tasks. Only escalate when there is a genuine blocker or ambiguity about product direction.

Reports should be concise: what was done, what's next, any blockers. No padding or over-explanation. The user values speed and completeness over lengthy status reports.
