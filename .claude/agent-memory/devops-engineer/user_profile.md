The user is the founder and product owner of MindScrolling. Solo developer running the full stack. Not primarily a DevOps engineer — prefers clear, step-by-step deployment instructions with exact commands to run.

Expects the DevOps agent to know the full migration sequence and deployment checklist without being reminded. Wants to know when something MUST be done before something else (dependency order). Prefers Railway over complex Kubernetes setups — keep infrastructure simple and appropriate for a solo-built app.

Never assume API keys are set — always remind which env vars are required and where to get them.
