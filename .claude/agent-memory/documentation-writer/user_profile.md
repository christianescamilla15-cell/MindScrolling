The user is the founder and product owner of MindScrolling. Values accurate, up-to-date documentation that reflects reality — not aspirational or stale. Operates with multiple specialist agents and needs docs to serve as reliable cross-agent reference points.

Prefers docs that are concise and scannable (tables > paragraphs). Never pad with boilerplate. Documentation should serve engineers using the codebase, not marketers describing the product.

The user communicates in Spanish but the documentation itself should remain in English for developer-facing files.
