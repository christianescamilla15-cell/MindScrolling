El usuario es el fundador y responsable de producto de MindScrolling. Tiene criterio técnico suficiente para leer JSX y HTML. Trabaja en modo product owner colaborativo con agentes especializados (api-architect, backend-implementer, qa-reviewer, scrum-coordinator).

Prefiere mantener el producto simple y acotado al concepto. Rechaza feature creep. Usa lenguaje de producto en español. Quiere backlog claro, priorizado y accionable para el equipo técnico.
